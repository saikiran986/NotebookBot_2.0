JavaScript Async & Promises Revision Handbook

1. How JavaScript Executes Code

Definition: JavaScript executes code synchronously, one statement at a time using a single Call Stack.

      |
      v
Call Stack

Execute -> Remove -> Execute Next

Why? JavaScript is single-threaded, meaning only one piece of JavaScript code executes at a time.

7. Promises

Definition: An object representing the eventual completion or failure of an asynchronous operation.

const promise = new Promise((resolve,reject)=>{
    resolve("Success");
});

States

  • Pending
  • Fulfilled
  • Rejected
Pending
 |
 +--> resolve() --> Fulfilled
 |
 +--> reject() --> Rejected

8. Promise Chaining

Promise.resolve(10)
.then(v=>v*2)
.then(v=>v+5)
.then(console.log);

Rule: Every .then() returns a new Promise.

return value → next then receives value.

throw error → nearest catch executes.

10. finally()

Runs regardless of success or failure.

Promise.resolve()
.finally(()=>{
   console.log("Cleanup");
});

Returning a value from finally() does NOT change the chain.

Throwing inside finally() rejects the chain.

11. Promise APIs

APIPurposeRejects?Returns
Promise.all()Wait for allFirst rejectionArray of values
Promise.allSettled()Wait for allNever due to input rejectionArray of status objects
Promise.race()First settled winsPossibleFirst settled result
Promise.any()First fulfilled winsOnly if all rejectFirst fulfilled value

Memory Tricks

  • all() → Everyone must succeed.
  • allSettled() → Collect everyone's result.
  • race() → First to settle wins.
  • any() → First success wins.

12. Interview Rules

  • Promise.then() callbacks are Microtasks.
  • setTimeout callbacks are Macrotasks.
  • Microtasks execute before Macrotasks.
  • Every then/catch returns a new Promise.
  • Promises cannot be automatically cancelled.
  • Promise.all() does not stop other Promises.
  • finally() is for cleanup, not changing values.
